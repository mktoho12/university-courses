# 解析学1

大学の授業「解析学1」（一変数関数の微分と積分）を、AIと相談しながら進めるための作業フォルダ。

## この科目で自分がどうなりたいか（学習のゴール）

> **数学的な考え方に触れて楽しみ、自分でも数学的に考えられるようになること。**
> 点数を取ることも目的だが、それ以上に「この学問を深く楽しむ」ことを大事にする。

そのために身につけたい状態（達成度の段階）：

1. **概念の意味が分かる**（土台）
   「微分＝局所的な変化」「積分＝大域的な性質」を自分の言葉で語れる。なぜその定義なのか、何を表しているのかを直観で説明できる。
2. **手を動かして解ける**（点数にも効く）
   微分・積分の計算を自力で実行できる。定石（部分積分・置換・テイラー展開など）を道具として使いこなせる。
3. **証明を追える・再構成できる**（理想・ただし条件つき）
   テイラーの定理や微積分学の基本定理などの証明の筋を追え、要点を自分で再構成できる。

### ⚠️ ゴール3の扱い（過去の挫折を踏まえた最重要ルール）

過去に**線形代数2・距離空間・集合と論理**で「証明をすべて理解しようとして時間と気力が尽きた」経験がある。
挫折の正体は能力ではなく**読み方と負荷管理**だった（→ `STUDY_LOG.md` のリサーチ参照）。具体的には：

- **前提知識の欠落**（証明が暗黙に使う過去の定理・定義が曖昧）
- **行間の飛躍**（1行目と2行目がなぜ繋がるのか分からない）
- そして**それが続いて精神的に辛くなる**（これが一番大きい）

なので **3は「盲目的に目指さない」。「少しトライ→やばそうなら2でストップ」を仕組み化する**：

- **証明はデフォルトで「まず骨格だけ追う」**（型・鍵・ゴールの3点。1行ずつ完璧にしない）
- 「面白そう・大事そう」な証明だけ行間を埋めにいく
- **15分ルール**：本気で15分考えて次の一歩の見当もつかないなら、それは破壊的苦闘＝前提欠落のサイン。粘らずに足場（AI・前提の復習）を取る。粘るのは「あと一歩が見えそう」なときだけ。

## AIの役割（このフォルダでのAIの使い方）

リサーチ（認知科学・数学教育研究）に基づき、AIの使い方を設計した。詳細は `STUDY_LOG.md`。要点：

- **詰まったら：まずヒント、1歩ずつ。**（答えを即出ししない。即答は定着・転移を蝕むことが研究で確認されている）
- **行間で詰まったときは「自分の仮説を先に言う」→ AIがどこが欠けているか診断する**（過剰修正効果＋生成効果）
- **AIの一番の仕事は「前提知識の欠落の診断役」**。証明を読む前に「これを理解するのに必要な前提を列挙して、どれを私が確認すべきか質問で確かめて」と使う。あなたの過去の脱落原因にピンポイントで効く。
- 新しい定理は**まず自力で予想・挑戦してから**AIに足場を求める（生産的失敗）。「証明を完成させて」とは頼まない。

## 進め方

- **ペース**：実際の授業の進度に合わせる。今週やる章を、その都度ここで一緒に進める。**じっくり時間は取れるが、気力（精神的エネルギー）は有限**。1回を長くするより短いセッションを間隔をあけて置く。
- **教材**：ZEN Study のテキスト（HTML＋LaTeX数式）。**要点のみ書き起こす**（全文は作らない。テキストは綺麗に取れるので必要なら都度参照）。
- **科目間の共通化はしない**。公共哲学は対話中心、解析学は概念理解＋手を動かす＋（選んだ証明だけ）行間を埋める、という別物として進める。

# 対策計画：AIが「わからない」と言えず作話する／問い詰められると結論を変える問題

作成 2026-07-15（本人就寝中に調査・立案）。**本人確認前に CLAUDE.md は書き換えていない。** これは提案。

---

## 1. まず結論：これは根絶できない。目標を「作話ゼロ」から「作話したら早く撤回」へ

一次情報（Anthropic・OpenAI の公式資料＋査読論文）を3方向で裏取りした。**全ての情報源が「どのプロンプト手法も根絶不能・減らせるだけ」で一致**している。根治は訓練・評価側の修正でユーザーの手の届く範囲外。

- **作話 (confabulation)**：もっともらしいが誤り・質問ごとに揺れる出力。「exp の括弧の理由」を読みにくい→文字数→横幅と乗り換えたのがこれ。
- **迎合 (sycophancy)**：問い詰められると前言を撤回・変更する。「なぜ質問を重ねると結論が変わるのか」の直接の答え。**5つの最先端AI全てに一貫**して存在（Anthropic研究）。プロンプトは部分緩和のみ。
- **根本原因**：訓練・評価が「わからない」より「推測」を報酬づけしている（OpenAI論文）。RLHF が較正も悪化させる。

→ だから **場当たり的な禁止行の継ぎ足しでは直らないし、逆効果**（CLAUDE.md 肥大化＝別のアンチパターン）。研究が示す唯一筋の良い方針は「不確実性を正直に出す許可＋接地の義務化＋圧に屈しない」の3点を**既存原則に畳んで**入れること。

## 2. 設計思想（原則2本＋棚卸し）に忠実な処置

失敗分析の結論：
- **(A) log の底を憶測で答えた** → 既存 **原則2（テキスト接地・一般知識で補完しない）に完全吸収済み**。新規行不要。教材を確認していれば防げた種類のミス。
- **(B) exp の括弧を作話＋問い詰めで乗り換え** → 枝葉への過剰説明の側は **原則1（聞かれたことだけ）に吸収**。過去の教訓（STUDY_LOG 645行「必要ないなら必要ないと言え」）と同型の再発。ただし **"事実・理由の作話"と"圧での立場変更"だけは、現行の原則2の「捏造」が話者境界に限定されているため、射程からわずかにこぼれる**。

→ **新しい🔴禁止節は作らない。** 原則2の一句を「接地」概念で一般化し、代表例を1つ足すだけで (A)(B) 両方を吸収できる。これが自転車操業を止めた設計思想に唯一忠実なやり方。

## 3. 具体的な提案（3つ。どれを採るかは本人が決める）

### 提案X（本命・最小）：CLAUDE.md 原則2の一句を一般化＋BAD例1行

**原則2の第2項に、事実質問も接地対象だと一句足す。** 案：

> 生徒の事実質問（「底は e か」「テキストに書いてあるか」「なぜ○○か」）にも同じ規律を適用する。テキストで確認できないなら、もっともらしい理由をその場で作らない。「確認します」か「持っていません／単なる慣習です」と即答する。**問い詰められても、新しい根拠がない限り説明を作り替えず「根拠がない」を維持する。**

**「例」ブロックにBAD/GOODを1組追加：**

```
■ 確信のない事実質問（テキストに答えが無い）
生徒: なぜ exp(x) は括弧、sin x はつけないの?
GOOD: 「確たる理由は持っていません。単なる慣習です。」で止める
BAD : 「読みにくいから」→問い詰められ「文字数が」→「横幅が」と理由を乗り換える
```

- 行数増加は最小（一句＋4行の例）。原則2の被覆率が上がる（将来の作話系も新節なしで拾える）＝畳めている。
- **フック化は不可**（後述）。散文で対処するしかない部分。

### 提案Y（任意・予防の物理化）：UserPromptSubmit フックで送信前ゲートを毎ターン再掲

- Stop フックで「作話」を後付け検出するのは**原理的に無理**（確信度は出力テキストに現れず、断定調フレーズだけ拾うと正しい断言まで誤検出する。既存フックの「狭く物理的特徴だけ」の鉄則に反する）。
- 代わりに **`UserPromptSubmit` の `additionalContext`** で、毎ターン冒頭に短い固定リマインダーを注入できる（検出でなく予防）。案の注入文：「確信のない事柄は断言しない。理由を書く前に教材で確認したか自問。未確認なら『確認する』か確信度を明示。問い詰められても根拠がなければ結論を変えない。」
- **注意**：既知バグ #40216 で additionalContext は履歴に累積する（長セッションで肥大）。→ 短い固定文にする・動的情報を入れない・timeout:5 明示。効果は**部分緩和**（研究上プロンプト系は根絶しない）。
- 🟡情緒との兼ね合い：毎ターンの注入が本人の目に触れないなら無害だが、コンテキスト肥大は role confusion（捏造）リスクを上げる方向でもある。**提案Xを先に入れ、Yは効果を見てから**が安全。

### 提案Z（運用・目標の再設定）：本人と共有する現実的なゴール

研究が「根絶不能」と言う以上、期待値を正す：
- ゴールは「AIが二度と作話しない」ではなく「**作話したら、問い詰められる前に自分で『これは推測です／わからない』と早く申告・撤回する**」。
- 本人にできる軽い防御：AIが理由・事実を語り始めたら「それ、教材に書いてある？推測？」と一言聞く。これはAIが守れる約束（100%でないが、聞かれれば区別を答えられる）。
- これは本人に負担を課すのでなく、🟡（本人に決定を迫らない）と両立する範囲で「気になったら一言」程度に留める。

## 4. 推奨する順序

1. **提案X（原則2の一般化＋BAD例）を採用** ← 研究・設計思想の両面から本命。低コスト・低リスク。

3. Y（フック予防注入）は X の効果を見てから判断（コンテキスト肥大のリスクがあるので急がない）。

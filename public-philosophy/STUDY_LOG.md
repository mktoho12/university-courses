## 教材へのアクセス方法（重要・再開時に使う）

ZEN Study（旧N高，`nnn.ed.nico`）の教材は要ログイン。
agent-browser の保存セッション `nnn` にログイン状態（cookie/localStorage）が保存済み。

### 通常運用（ヘッドレスで取得）

**重要：教材ページは PowerPoint スライドが画像（PNG）で並んでいるだけ。**
本文テキスト抽出（get text）だと概要しか取れない。スライド画像をDLして読む。

```bash
# 1) ページを開く（--session-name nnn でログイン状態を自動復元）
agent-browser --session-name nnn open "<教材URL>"
agent-browser --session-name nnn wait 2500           # SPAの描画待ち

# 2) スライド画像のURLを抜き出す（1920x1080 PNG, cdn-private.nnn.ed.nico）
agent-browser --session-name nnn eval \
  "JSON.stringify(Array.from(document.querySelectorAll('img')).map(i=>i.src).filter(s=>s.includes('cdn-private')))"

# 3) curlでDL（署名付きURLなのでcookie不要・そのまま200で取れる。有効期限あり=取得は都度）
curl -s -o slideNN.png "<署名付きURL>"

# 4) Readツールで画像として読む（OCR的に内容を理解）→ material.md に書き起こす
```

- 署名付きURL（Policy/Signature/Key-Pair-Id 入り）は **期限切れ**になるので、
  古いURLは使い回さず、開くたびに eval で取り直す。
- `get text 'body'` / `snapshot -c` はページ概要・見出しの確認用に併用。

- `--headed` も `--executable-path` も付けない＝ヘッドレス本番運用。
- 認証情報は 1Password CLI（`op`）に保存。項目は **`op://Personal/ZEN ID`**
  （username=メール / password）。`auth.zenid.jp` 経由のログイン。

### セッションが切れた（再ログインが必要）になったら

1. ヘッド付きでログインページを開く（クリーンな実機Chromeを使う）:
   ```bash
   agent-browser --session-name nnn --headed \
     --executable-path "/Applications/Google Chrome.app/Contents/MacOS/Google Chrome" \
     open https://www.nnn.ed.nico/login
   ```
   ※ chrome-headless-shell だと窓が出ないので、必ず実機Chromeを指定。
2. メール→次へ→パスワード→ログイン。値は `op read "op://Personal/ZEN ID/username"`
   / `.../password` で取得し、`agent-browser ... fill '@ref' "$VALUE"` で流し込む
   （画面・ログに出さない）。op はデスクトップアプリ連携（Touch ID承認）。
3. `https://www.nnn.ed.nico/home` に着いたら `close` でセッション確定。
